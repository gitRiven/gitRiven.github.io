事件详解

事件注册

  • on注册事件

使用on注册事件不能给一个元素注册多个事件

  • addEventListener(参数1,参数2,参数3)
    • 参数1 要注册的事件名 : click mousemove
    • 参数2 事件处理程序,可以给一个匿名函数 也可以给一个事先定义好的函数名
    • 参数3 布尔值 ,默认为false false 代表支持冒泡,如果为true则为支持捕获
      • 这个注册事件的方式，ie8及以前不支持
  • ie8及以前
  • 元素名.attachEvent(参数1,参数2);
    • 参数1: 事件类型 带on的。 例如：onclick onmousemove….
    • 参数2：事件处理程序，可以给一个匿名函数，也可以给一个事先定义好了的函数名。

注册事件做兼容

谷歌支持addEventListener，输出就会得到这个事件的函数体，ie8及以前不支持，就返回一个undefined。
ie8及以前支持attachEvent，就返回一个函数体。

移除事件

  • 移除on绑定的事件 obj.onclick = null
  • 移除addEventListener注册的事件
  • 移除.attachEvent注册的事件

移除事件做兼容

target 可以获取触发事件的Dom节点

事件的冒泡

  • 事件冒泡：如果一个元素的某个事件被触发了，那么他的所有的父级元素，同样的事件都会被触发。子元素没有绑定事件也会触发父元素相应的事件

为什么以前没有？ 事件冒泡一直存在，以前看不见，是因为没有给触发事件的元素的父元素设置同样的事件。

  • 阻止事件冒泡
    e.stopPropagation(); //ie8及以前不支持。
    e.cancelBubble = true; //ie8及以前支持

事件的捕获

事件捕获： 先触发最外层的拥有相同事件的父级元素的这个事件，然后一层层的往里触发，直到事件源为止。

事件的三个阶段

e.eventPhase 就可以获取到这个事件触发的是那个阶段。会返回一个1-3的值分别对应三个阶段
1.捕获阶段
2.正常阶段
3.冒泡阶段
